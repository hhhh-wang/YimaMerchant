

酒店B2B合作平台后台系统功能规划 (基于若依框架)
项目概述
系统名称：酒店B2B合作平台后台管理系统
技术架构：若依前后端分离版本 (RuoYi-Vue)
核心目标：为平台BD、酒店运营、客服、财务等角色提供完整的酒店签约、运营、交易、结算全流程管理能力

菜单如下：

````
[酒店合作中心]   (面向：BD、运营)
├─ 待签约酒店
├─ 合作酒店
├─ BD-酒店归属管理 (核心交接功能在此)
└─ (可选)BD绩效看板

[酒店运营中心]   (面向：酒店运营、酒店方)
├─ 酒店信息
├─ 房型管理
└─ 库存价格管理

[交易与服务中心] (面向：客服、运营)
└─ 酒店订单

[财务结算中心]   (面向：财务)
└─ 账单管理

````



模块一：酒店合作中心
面向角色：BD商务经理、运营管理人员
核心职责：酒店的引入、签约、商务关系管理与业绩跟踪
1.1 待签约酒店
页面路由：/cooperate/pendingList
功能描述：处理酒店入驻申请与资质审核流程
核心功能清单：
[ ] 列表查询页面：
表格展示字段：酒店名称、申请状态（待审核/已通过/已驳回）、申请人、申请时间、最后操作时间
搜索条件：酒店名称、申请状态、申请时间范围
分页组件：标准若依分页
操作列：查看、审核、删除
[ ] 酒店申请表单（弹窗/抽屉）：
基础信息：酒店名称、联系方式、联系人
资质上传：营业执照、特种行业许可证、卫生许可证（支持多图上传）
地理位置：省市区选择、详细地址、地图坐标（集成地图组件）
申请备注：多行文本输入框
[ ] 审核流程：
审核通过：生成正式合作记录，自动创建酒店后台账号
审核驳回：填写驳回理由，状态变更为"已驳回"
审核日志：记录所有审核操作的时间、人员和备注
[ ] 批量操作：
批量导出申请列表（Excel）
批量删除无效申请
后端接口规划：
GET    /cooperate/pending/list          # 待签约酒店列表
GET    /cooperate/pending/{id}         # 获取申请详情
POST   /cooperate/pending              # 新增申请
PUT    /cooperate/pending/approve      # 审核通过
PUT    /cooperate/pending/reject       # 审核驳回
DELETE /cooperate/pending/{ids}        # 删除申请
POST   /cooperate/pending/export      # 导出申请列表

1.3 BD-酒店归属管理
页面路由：/cooperate/bdHotelBind
功能描述：管理BD与酒店的绑定关系，支持一键交接
核心功能清单：
[ ] 归属关系列表：
表格字段：BD姓名、BD工号、酒店名称、绑定时间、绑定状态
双向筛选：按BD筛选其名下酒店 / 按酒店筛选当前BD
状态标识：正常绑定、已解绑（历史记录）
[ ] BD选择器组件：
下拉选择框，仅显示在职状态的BD
支持搜索BD姓名/工号
显示BD当前负责酒店数量
[ ] 酒店选择器组件：
支持按酒店名称搜索
支持多选模式
显示酒店当前绑定BD（如有）
[ ] 绑定/解绑操作：
单个绑定：选择BD + 选择酒店 → 确认绑定
批量绑定：选择BD + 选择多个酒店 → 批量绑定
单个解绑：确认解绑关系
批量解绑：批量解除选中酒店的绑定
[ ] 一键交接功能：
场景1：BD离职交接
在BD管理页禁用该BD账号
系统提示"检测到BD[姓名]已离职，是否立即转移其名下酒店？"
选择接收BD，一键完成所有酒店归属转移
场景2：业务调整批量转移
筛选出源BD名下所有/部分酒店
批量选择 → 批量更换BD → 选择目标BD
[ ] 绑定历史记录：
记录所有绑定/解绑/变更操作
字段：操作时间、操作人、原BD、新BD、酒店、操作类型
后端接口规划：
GET    /cooperate/bind/list            # 归属关系列表
POST   /cooperate/bind/batch          # 批量绑定
PUT    /cooperate/bind/transfer       # 批量转移
DELETE /cooperate/bind/{ids}          # 批量解绑
GET    /cooperate/bind/history        # 绑定历史记录
GET    /cooperate/bind/bdOptions      # 获取BD下拉选项
GET    /cooperate/bind/hotelOptions   # 获取酒店下拉选项

2.3 库存价格管理
页面路由：/hotel/inventoryPrice
功能描述：酒店房型的每日价格、库存、退款规则设置（核心运营功能）
核心功能清单：
[ ] 双层选择器：
第一层：酒店选择下拉框（选择后加载该酒店下所有房型）
第二层：房型选择下拉框（支持多选，用于批量设置）
[ ] 日历视图（核心交互）：
月视图展示，默认显示当前月
支持月份快速切换
房态颜色标识：可售（绿色）、关房（红色）、库存紧张（橙色）
[ ] 日历表格列（动态根据佣金模式变化）：
底价模式列：

| 日期 | 星期 | 结算底价 | 平台卖价 | 佣金 | 库存 | 退款规则 | 操作 |
| ---- | ---- | -------- | -------- | ---- | ---- | -------- | ---- |
|      |      |          |          |      |      |          |      |
卖价模式列：
| 日期 | 星期 | 酒店卖价 | 佣金比例 | 佣金 | 库存 | 退款规则 | 操作 |
| ---- | ---- | -------- | -------- | ---- | ---- | -------- | ---- |
|      |      |          |          |      |      |          |      |
公共列：日期、星期、库存、退款规则、操作
[ ] 单日编辑弹窗：
根据酒店佣金模式动态显示字段
底价模式字段：结算底价、平台卖价、库存
卖价模式字段：酒店卖价、库存（佣金比例从合同读取，只读显示）
退款规则：下拉选择（限时退/不可退/任意退）
特殊标记：可标记为节假日/特殊日期
保存并应用到：仅当天/同星期几/同房型所有日期
[ ] 批量操作功能：
批量选择日期（可框选、可多选）
批量设置弹窗：价格、库存、退款规则
批量关房：将选中日期库存设为0
批量复制：复制某天设置到其他日期
批量恢复默认：恢复为房型默认设置

后端接口规划：
GET    /hotel/price/calendar      # 获取价格日历（酒店+房型+日期范围）
POST   /hotel/price/daily        # 设置单日价格库存
POST   /hotel/price/batch        # 批量设置价格库存
GET    /hotel/price/history      # 价格修改历史

4.1 账单管理
页面路由：/finance/billList
功能描述：酒店结算账单的生成、核对、确认与支付
核心功能清单：
[ ] 账单列表：
表格字段：账单号、酒店名称、结算周期、订单数量、结算金额、状态、生成时间
状态标签：待生成/待确认/已确认/付款中/已付款/有争议
筛选条件：酒店、状态、结算周期、生成时间
金额汇总：列表顶部显示当前筛选条件下的总金额
[ ] 账单生成：
手动生成：选择酒店 + 结算周期（如2025-10-01至2025-10-31）
自动生成：系统按配置自动生成周期账单（可配置每月1日生成上月账单）
生成规则：基于订单状态（只结算"已完成"订单）、佣金模式计算
生成预览：生成前预览账单明细，确认无误后正式生成
[ ] 账单详情页（新页面）：
账单摘要卡片：
结算周期、生成时间
订单总数、间夜总数
总房费、总佣金、应付金额
订单明细表格：
列表显示该账单包含的所有订单
字段：订单号、入住日期、房型、订单金额、佣金、结算金额
支持按订单号搜索
可展开查看单笔订单详情
操作按钮区（根据状态动态显示）：
待确认：酒店可在线确认
已确认：财务可发起付款
有争议：可重新核对生成
[ ] 对账流程：
酒店端：登录后台查看账单，核对明细，在线确认或提出异议
财务端：查看酒店确认状态，处理异议，重新生成
对账记录：记录每次对账操作
[ ] 付款管理：
付款申请：财务提交付款申请
付款审批：多级审批流程（可配置）
付款记录：记录付款时间、付款单号、凭证
发票管理：关联酒店开票信息
[ ] 账单导出：
导出账单汇总（PDF/Excel）
导出订单明细（Excel）
打印对账单
后端接口规划：
GET    /finance/bill/list             # 账单列表

GET    /finance/bill/{billNo}        # 账单详情
GET    /finance/bill/orders          # 账单订单明细

PUT    /finance/bill/dispute         # 提出异议

POST   /finance/bill/payment         # 发起付款
GET    /finance/bill/export          # 导出账单
